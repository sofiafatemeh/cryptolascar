# Milestone v1.1: Rapports Enrichis

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 6–8
**Total Plans:** 11

## Overview

Milestone v1.1 transforms CryptoLascar from plain-text email reports into visually rich financial reports. Phase 6 built the chart generation module (matplotlib PNG charts embedded as base64, with graceful fallback per chart). Phase 7 redesigned the HTML email templates with a dark-mode Bloomberg aesthetic, mobile responsiveness, and wired all three report types to the new visual system. Phase 8 (inserted after a milestone audit revealed production charts were all broken) fixed three data-contract mismatches between v1.0 collectors and v1.1 chart generators, added 7-day sparkline collection, fixed the Fear & Greed unguarded AttributeError, and wrote Phase 6's missing VERIFICATION.md.

## Milestone Summary

**Inserted Phases:**
- Phase 8: Close Gaps — inserted after Phase 7 after milestone audit revealed 3 broken chart E2E paths

**Key Decisions:**
- 08-01-A: `_fetch_1w_pct` separated from `_fetch_yfinance` — same Ticker, independent `history` call, result added before cache upsert
- 08-01-B: Sparklines enriched even on cache batch hit — sparkline cache (coingecko_sparkline) checked separately with 1h TTL
- 08-01-C: `history=[]` on market_chart failure — `generate_crypto_sparklines` returns None (existing fallback)
- 08-02-A: `_PEA_NAMES`/`_PEA_ELIGIBILITY` defined at module level in all 3 reporters — config/logic separation
- 08-02-B: `etf_chart_data` filters `pct_change is not None` — CHART-05 tamper filter
- 08-02-C: `pea_list` filters `price is not None` — avoids empty rows in `generate_pea_table`
- 08-02-D: `_crypto_section` in daily.py — `fg = crypto.get('fear_greed', {}) or {}` (Rule 1) — was returning None if value=None

**Issues Resolved:**
- CHART-01: ETF bar chart never rendered in production — dict shape mismatch + missing 1w data
- CHART-02: Crypto sparklines never rendered — sparkline=false in CoinGecko call, no 7d history
- CHART-03: Full report collapse when fear_greed=None — unguarded None.get('value') outside per-generator try/except
- CHART-04: PEA table never rendered — full dict passed instead of list[dict]
- Test suite masked all 3 integration blockers — all reporter tests mocked chart generators with return_value=None

**Technical Debt Incurred (carried to v1.2):**
- CR-01: ETF chart title uses ISO date format, not French localized (cosmetic)
- CR-02: Markdown tables in section bodies render as literal pipe text in weekly/monthly HTML (white-space:pre-wrap)
- IN-01/02: `_build_chart_panel()` and `_sections_to_html()` duplicated verbatim across 3 reporters
- SCHED-03: Crontab offset divergence (install_cron.sh: `1 8 * * *` vs installed: `0 8 * * *`) — human decision needed
- WR-01: main.py locale.Error catch is silent — monthly subject may be English with no warning
- WR-02: collect_all() runs before monthly guard — wastes API quota on non-last-days for --mode monthly
