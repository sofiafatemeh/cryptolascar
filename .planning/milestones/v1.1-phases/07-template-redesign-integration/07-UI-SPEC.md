# Phase 7 — UI Design Contract: Template Redesign & Integration

> Visual and interaction contract for HTML email template redesign.
> This is NOT a web frontend. All design tokens are expressed as inline CSS.
> Layout is table-based for email-client compatibility (no CSS grid, no flexbox).
> Color tokens MUST match Phase 6 UI-SPEC exactly.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Jinja2 + HTML/CSS (inline) |
| Preset | not applicable — email template, no shadcn, no npm |
| Component library | none — table-based HTML layout only |
| Icon library | none |
| Font | `'Courier New', monospace` — everywhere (headings, body, data values, footer) |
| Rendering target | Gmail (desktop + mobile), standard email clients |
| Template file | `templates/report_email.html` (single universal template) |
| Template engine | Jinja2 with `autoescape=True`; `body_html` passed with `| safe` filter |

> Source: D-06, D-07, D-09 from 07-CONTEXT.md; existing `delivery/email.py` Jinja2 setup.

## Spacing Scale

Declared as inline CSS pixel values. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline spacing |
| sm | 8px | Section card bottom margin; PEA table cell vertical padding |
| md | 16px | Section card inner padding; chart image top/bottom margin |
| lg | 24px | Header band vertical padding (top + bottom) |
| xl | 32px | Outer email wrapper top/bottom padding |
| 2xl | 48px | Major visual break between header band and chart panel |
| 3xl | 64px | Not used in this phase |

**Applied values:**

| Element | Property | Value |
|---------|----------|-------|
| Outer wrapper (`<table>` max-width wrapper) | padding | 32px 24px |
| Header band (`<td>` containing branding) | padding | 24px 24px |
| Section card (`<div>` per narrative section) | padding | 16px |
| Section card | margin-bottom | 8px |
| Chart panel outer container | padding | 16px 0 |
| Chart `<img>` | margin | 16px 0 |
| Chart panel between pairs (2×2 layout) | cell-spacing | 8px |
| Footer `<p>` | margin-top | 24px |
| Footer `<p>` | padding-top | 16px (above hr separator) |

Exceptions: Touch-target minimum 44px does not apply — email templates have no interactive tap targets.

> Source: D-10 (section card 16px padding, 8px bottom margin), D-07 (Bloomberg terminal structure),
> Phase 6 UI-SPEC IMG tag template (margin:16px 0), existing template (padding:32px 24px).

## Typography

All roles use `'Courier New', monospace` — no role uses a different font family (D-09).
All sizes are in px for email rendering. Line heights are expressed as unitless ratios.

| Role | Size | Weight | Line Height | Color | Usage |
|------|------|--------|-------------|-------|-------|
| Header brand (`CryptoLascar`) | 22px | 700 (bold) | 1.2 | `#FF6B35` | Left side of header band |
| Header badge (`[DAILY]` etc.) | 14px | 700 (bold) | 1.2 | `#FF6B35` | Right side of header band, report type |
| Header date | 12px | 400 (regular) | 1.2 | `#e0e0e0` | Right side of header band, below badge |
| Section heading (`h2`) | 18px | 700 (bold) | 1.2 | `#FF6B35` | Section titles inside cards |

| Footer disclaimer | 12px | 400 (regular) | 1.5 | `#555555` | Bottom of email |

| Role | Mobile Size | Rationale |
|------|-------------|-----------|
| Body prose | 14px minimum | TMPL-02 readable on mobile |

## Template Variables Contract

Variables passed to `report_email.html` by `send_email()`:

| Variable | Type | Example | Auto-escaped |
|----------|------|---------|--------------|
| `subject` | `str` | `"[DAILY] Analyse du 2026-05-14"` | Yes |
| `report_type` | `str` | `"daily"` / `"weekly"` / `"monthly"` | Yes |
| `date` | `str` | `"2026-05-14"` | Yes |
| `body_html` | `str` (pre-built HTML) | Rich HTML with `<img>` and PEA table | No — `| safe` filter |

`body_html` contains the full content block: charts (as `<img>` tags or fallback `<p>` tags) + narrative sections (pre-rendered `<div>` cards from `_markdown_to_html()` or reporter HTML builder). Passed with `| safe` because it is pre-escaped by the reporter layer.

Header badge mapping (driven by `report_type` value):

| `report_type` | Badge string |
|---------------|-------------|
| `daily` | `[DAILY]` |
| `weekly` | `[WEEKLY WRAP]` |
| `monthly` | `[MONTHLY CLOSE]` |

> Source: D-06 (one universal template with Jinja2 conditionals), D-08 (template variables
> report_type and date), existing `delivery/email.py` build_subject() + template render pattern.

## Layout Structure

### Outer Wrapper

```
<body background="#0d0d0d">
  <table width="100%" max-width:680px, margin:0 auto>   ← outer centering wrapper
    <tr><td>
      [HEADER BAND]
      [CHART PANEL]
      [NARRATIVE SECTIONS]
      [FOOTER]
    </td></tr>
  </table>
</body>
```

Desktop: max-width 680px, centered.
Mobile (`≤600px`): single-column, full width, font-size ≥ 14px, no horizontal overflow.

### Header Band

Full-width dark band (`background:#111111`) spanning the full 680px container width.
Two-column table inside: left cell = brand name, right cell = badge + date.

```
| CryptoLascar (orange, 22px bold) |  [DAILY] (orange, 14px bold)  |
|                                   |  2026-05-14 (grey, 12px)      |
```

On mobile (≤600px): stack vertically (brand on top, badge+date below), both centered.

### Chart Panel

Positioned immediately after the header band, before any narrative section (D-11).
Contains all 4 chart elements for all 3 report types (D-12, D-13, D-14).

**Desktop layout — 2×2 table grid:**

```
| ETF bar chart (CHART-01)          | Crypto sparklines (CHART-02)      |
| Fear & Greed gauge (CHART-03)     | PEA colored table (CHART-04)      |
```

Each cell: `width="50%"`, `padding:8px`, `vertical-align:top`.
Chart images use the Phase 6 IMG tag template exactly:

```html
<img src="data:image/png;base64,{b64_string}"
     alt="{alt_text}"
     style="display:block;max-width:100%;height:auto;margin:16px 0;" />
```

**Mobile layout (≤600px):** 4 stacked rows, each chart full width (single-column).

**Chart fallback (CHART-05 / D-15):** when chart function returns `None`, render the Phase 6 fallback string in the corresponding cell:

| Chart | Fallback HTML (from Phase 6 UI-SPEC — use exactly) |
|-------|-----------------------------------------------------|
| ETF bar chart | `<p style="color:#888;font-style:italic;">[Graphique ETF indisponible]</p>` |
| Crypto sparklines | `<p style="color:#888;font-style:italic;">[Graphique crypto indisponible]</p>` |
| Fear & Greed gauge | `<p style="color:#888;font-style:italic;">[Gauge Fear &amp; Greed indisponible]</p>` |
| PEA table | `<p style="color:#888;font-style:italic;">[Tableau PEA indisponible]</p>` |

### Narrative Sections

Each narrative section (Macro Snapshot, ETF Radar, Crypto Pulse, PEA Alert, News Feed, One Signal for daily; analogous sections for weekly/monthly) is wrapped in a section card:

```html
<div style="background:#111111;border:1px solid #2a2a2a;padding:16px;margin-bottom:8px;">
  <!-- section heading and body rendered by _markdown_to_html() or reporter HTML helper -->
</div>
```

Section headings (h2) rendered as:
```html
<h2 style="color:#FF6B35;font-family:'Courier New',monospace;font-size:18px;
           font-weight:700;margin-top:0;margin-bottom:12px;line-height:1.2;">
  {heading text}
</h2>
```

Body paragraphs rendered as:
```html
<p style="color:#e0e0e0;font-family:'Courier New',monospace;font-size:14px;
          line-height:1.6;margin:0 0 12px 0;">
  {paragraph text}
</p>
```

### Footer

Separated from content by a 1px solid `#2a2a2a` horizontal rule:
```html
<hr style="border:none;border-top:1px solid #2a2a2a;margin:24px 0;">
<p style="font-size:12px;color:#555555;font-family:'Courier New',monospace;
          font-style:italic;margin:0;">
  Ceci n'est pas un conseil financier. Informations à titre éducatif uniquement.
</p>
```

> Source: D-07 (Bloomberg terminal feel), D-10 (section card geometry), D-11 (chart panel at top),
> D-12/D-13/D-14 (all 4 charts in all 3 report types), D-15 (fallback strings),
> TMPL-02 (mobile single-column), D-19 (footer #555555), Phase 6 IMG tag template.

## Reporter Return Type Contract

Each reporter's build function returns a structure with two fields (D-02):

| Field | Type | Content |
|-------|------|---------|
| `html_body` | `str` | Rich HTML: chart `<img>` tags + PEA table HTML + section `<div>` cards |
| `plain_text` | `str` | Unchanged Markdown narrative for MIME text/plain fallback |

`html_body` is passed as `body_html` to the Jinja2 template with `| safe` filter.
`_markdown_to_html()` is bypassed when `html_body` is present (D-03).
`plain_text` feeds the `text/plain` MIME part — RFC 2822 compliance preserved (D-05).

> Source: D-02, D-03, D-05 from 07-CONTEXT.md.

## Email Client Safety (replaces Registry Safety)

This section replaces the shadcn registry gate. No npm, no shadcn, no component registry.

| Constraint | Rule |
|------------|------|
| Layout | Table-based only — no `display:flex`, no `display:grid` |
| CSS positioning | No `position:absolute/fixed/sticky` |
| CSS variables | Not used — inline literal values only (email client compatibility) |
| External resources | None — no web fonts, no external CSS, no CDN links |
| JavaScript | None — email clients block JS |
| Media queries | Permitted inside `<style>` block in `<head>` only; inline CSS is the fallback |
| `<style>` block scope | Mobile breakpoint overrides only (`@media (max-width:600px)`) |
| Base64 images | Inline `data:image/png;base64,` only — no `<img src="https://...">` |
| Font loading | `'Courier New', monospace` — system font, no `@font-face`, no Google Fonts |
| Jinja2 autoescape | `autoescape=True` on Jinja2 env — `body_html` uses `| safe` (pre-escaped by reporter) |
| Template variables | `subject`, `report_type`, `date` are auto-escaped; never use `| safe` on them |
| Path traversal | `_SAFE_TYPE_RE` + `_SAFE_DATE_RE` validation in `archive_report()` must be preserved |

> Source: D-07 (Bloomberg terminal, email-client safe), TMPL-02 (mobile responsiveness),
> existing `delivery/email.py` security patterns (T-04-01, T-04-02, CR-01, WR-01).
