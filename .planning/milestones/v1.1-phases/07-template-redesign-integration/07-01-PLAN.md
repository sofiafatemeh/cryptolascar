<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reporter-generated HTML → template body_html slot | html_body string built by reporters is marked safe with `| safe` — must be pre-escaped by reporters, never contain user-supplied raw strings |
| template variables (subject, report_type, date) → Jinja2 render | Auto-escaped by autoescape=True — never use `| safe` on these |
| html_section() title parameter → HTML output | Title must be html.escape()-d inside html_section() before insertion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Injection | `templates/report_email.html — {{ body_html \| safe }}` | mitigate | html_body is pre-escaped by reporter layer (reporters call html.escape() on user-visible strings via _markdown_to_html() and html_section()); template uses autoescape=True for all other variables; `| safe` only on html_body |
| T-07-02 | Injection | `reporters/base.py html_section(title)` — XSS via title parameter | mitigate | html_section() calls `_html_stdlib.escape(title)` on title before inserting into &lt;h2&gt; — raw title never reaches HTML output |
| T-07-03 | Injection | `templates/report_email.html — {{ date }}` and `{{ report_type }}` | accept | Both variables are auto-escaped by Jinja2 autoescape=True (select_autoescape(['html'])); production values are ISO date strings and fixed strings ("daily"/"weekly"/"monthly") — no user input path |
| T-07-04 | Tampering | Jinja2 autoescape bypass — `| safe` used on wrong variable | mitigate | Template must only use `| safe` on `body_html`; `subject`, `report_type`, `date` are never marked safe — enforced by acceptance_criteria grep check |
</threat_model>
