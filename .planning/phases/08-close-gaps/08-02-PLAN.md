---

**FIX 1 — CHART-03: fg_score line (1 line change, same in all 3 files)**

Find (exact current code):
```python
    fg_score = (data.get("crypto") or {}).get("fear_greed", {}).get("value")
```

Replace with:
```python
    fg_score = ((data.get("crypto") or {}).get("fear_greed") or {}).get("value")
```

Rationale: When `fear_greed=None`, the original `.get("fear_greed", {})` returns `None` (the default `{}` is only used when the key is absent, not when the value is `None`). Adding `or {}` after `.get("fear_greed")` ensures `None` is coerced to `{}` before `.get("value")` is called.

---

**FIX 2 — CHART-01: ETF data transform (replace the etf try block)**

Find the current ETF try block (exact code):
```python
    try:
        etf_b64 = generate_etf_chart(etf_data, date_str)
    except Exception:
        etf_b64 = None
```

Replace with (this block + the lines that set up `etf_chart_data` before the try):

```python
    etf_raw = data.get("etf") or {}

        sym: {
            "1d": t.get("pct_change") or 0.0,
            "1w": t.get("pct_change_1w") or 0.0,
        }
```
