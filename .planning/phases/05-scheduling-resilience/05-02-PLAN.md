<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| bash script→crontab | install_cron.sh modifies the user crontab — runs under the VPS user account |
| cron→filesystem | cron cd's to PROJECT_DIR and launches Python — PROJECT_DIR is set at script time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering | install_cron.sh PROJECT_DIR variable | mitigate | PROJECT_DIR derived from script's own directory (BASH_SOURCE[0]) — not taken from environment or user input; idempotent check prevents duplicate installs |
| T-05-06 | Elevation of Privilege | crontab install | accept | crontab runs as the current user (non-root); no sudo needed for user crontab; acceptable for a personal VPS |
| T-05-07 | Information Disclosure | smoke test output | mitigate | smoke test instructions tell user to check inbox directly — no credentials echoed in commands; run_log query only reads status/sources/error_msg fields |
</threat_model>
