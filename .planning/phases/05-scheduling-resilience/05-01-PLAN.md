<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cron→main.py | Cron fires main.py as a subprocess; no user input, but --mode is fixed in crontab |
| main.py→delivery | main.py passes report_type/date/report_text to send_email/write_tweet/archive_report |
| main.py→run_log | error_msg written to SQLite must never contain SMTP password or API keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | main.py outer try/except | mitigate | Log only report_type + str(exc) in error_msg; never include smtp_password, anthropic_api_key, or any credential in the err_msg string written to run_log |
| T-05-02 | Denial of Service | monthly cron fires daily | accept | by design (D-02/D-03): Python guard exits 0 in < 1s on non-last-day; negligible resource cost |
| T-05-03 | Tampering | report_type in pipeline loop | mitigate | report_type values come from select_reports() which returns only hardcoded keys {"daily","weekly","monthly"} — no external input reaches archive_report/send_email path |
| T-05-04 | Repudiation | run_log entries | mitigate | every run writes exactly one run_log entry (success/partial/skipped/error) — D-11 enforced in both success and exception paths |
</threat_model>
