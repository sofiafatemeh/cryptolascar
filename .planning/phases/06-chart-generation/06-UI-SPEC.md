# Phase 6 — UI Design Contract: Chart Generation

> Visual contract for Python matplotlib chart generation. This is NOT a web frontend.
> Charts are rendered as PNG, base64-encoded, and embedded inline in HTML emails.
> All design decisions must be forward-compatible with the Phase 7 dark-mode Bloomberg template.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | matplotlib (Python) |
| Preset | not applicable — Python/matplotlib project, no shadcn |
| Component library | none — charts are programmatic PNG, not HTML components |
| Icon library | none |
| Font (chart text) | DejaVu Sans (matplotlib default) — acceptable for phase 6; override to monospace in phase 7 if feasible |
| Output format | PNG via `io.BytesIO`, encoded as base64 UTF-8 string |
| Rendering backend | `matplotlib.use("Agg")` — non-interactive, VPS-safe |
| DPI | 150 — balances file size vs. sharpness at typical email widths |

## Chart Dimensions (replaces Spacing Scale)

Each chart type declares its own `figsize` and internal padding. All values are in matplotlib inches at 150 DPI.

### ETF Bar Chart (CHART-01)

| Property | Value | Rationale |
|----------|-------|-----------|
| figsize | (8, 4) | 1200 x 600 px at 150 DPI — wide enough to label all tracked ETFs |
| left margin | 0.12 (axes fraction) | Room for y-axis tick labels |
| right margin | 0.02 | Minimal dead space |
| top margin | 0.15 | Chart title clearance |
| bottom margin | 0.18 | Two-line x-axis ticker labels |
| bar width | 0.35 | Side-by-side bars (1-day + 1-week) with 0.05 gap between pairs |
| group spacing | 1.0 | One unit per ETF group |
| bar edge width | 0 | Flat fill, no border |

### Crypto Sparklines (CHART-02)

| Property | Value | Rationale |
|----------|-------|-----------|
| figsize | (8, 3) | 1200 x 450 px — compact, two curves |
| left margin | 0.10 | Minimal y-axis labels (price in USD abbreviation) |
| right margin | 0.02 | |
| top margin | 0.15 | Title + legend |
| bottom margin | 0.12 | Date x-axis |
| line width | 2.0 pt | Legible at email render size |
| marker size | 0 | No point markers — clean line only |

### Fear & Greed Gauge (CHART-03)

| Property | Value | Rationale |
|----------|-------|-----------|
| figsize | (5, 3) | 750 x 450 px — square-ish, self-contained |
| left margin | 0.05 | Minimal — arc is centered |
| right margin | 0.05 | |

| bottom margin | 0.05 | Score text below needle |
| arc thickness | 0.25 (fraction of radius) | Visible band without being chunky |
| needle width | 2.5 pt | Crisp pointer |

### PEA Colored Table (CHART-04)

> The PEA table is rendered as an HTML string, not a PNG. No figsize applies.

| Property | Value | Rationale |
|----------|-------|-----------|
| Output type | HTML string | Requirements.md CHART-04 specifies "tableau HTML coloré" |
| Cell padding | 8px vertical, 12px horizontal | Readable inline in email body |
| Row height | natural (content-driven) | No fixed height — content sets row height |
| Border | 1px solid #1a1a2e | Subtle separator, dark-mode compatible |

The arc is divided into five color zones corresponding to the standard Fear & Greed scale.

| Zone | Range | Color | Label |
|------|-------|-------|-------|
| Extreme Fear | 0–24 | `#FF4444` | Extreme Fear |
| Fear | 25–44 | `#FF8C42` | Fear |

| Greed | 56–74 | `#00C851` | Greed |

### 60/30/10 Summary

| Role | Value | Coverage |
|------|-------|----------|
| Dominant (60%) | `#0d0d0d` | Chart backgrounds, figure area |
| Secondary (30%) | `#1a1a2e` / `#1f1f1f` | Grid lines, table header, axis spines |
| Accent (10%) | `#FF6B35` (orange) + `#00C851` (green) | BTC line, table header text, positive bars, positive performance |

Accent reserved for: BTC sparkline, PEA table header text, positive performance bars, positive performance percentage values. NOT used for generic labels, titles, or axis ticks.

Red `#FF4444` is reserved exclusively for: negative performance bars, negative performance percentage values, Extreme Fear gauge zone.

### Fallback Text (CHART-05)

When any chart function returns `None` due to an exception, the email template must render these inline text replacements. These strings are part of the chart module's public interface (returned alongside `None` as a tuple or via a constant).

| Chart | Fallback HTML string |
|-------|----------------------|
| ETF bar chart | `<p style="color:#888;font-style:italic;">[Graphique ETF indisponible]</p>` |
| Crypto sparklines | `<p style="color:#888;font-style:italic;">[Graphique crypto indisponible]</p>` |
| Fear & Greed gauge | `<p style="color:#888;font-style:italic;">[Gauge Fear &amp; Greed indisponible]</p>` |
| PEA table | `<p style="color:#888;font-style:italic;">[Tableau PEA indisponible]</p>` |

### IMG Tag Template (for PNG charts)

When a chart function succeeds and returns a base64 string, the email template embeds it as:

```html
<img src="data:image/png;base64,{b64_string}"
     alt="{alt_text}"
     style="display:block;max-width:100%;height:auto;margin:16px 0;" />
```

Alt text per chart:
- ETF bar chart: `"Graphique de performance ETFs"`
- Crypto sparklines: `"Sparklines BTC et ETH sur 7 jours"`
- Fear & Greed gauge: `"Gauge Fear and Greed Index"`

## Graceful Degradation Contract (CHART-05)

This is a visual contract rule, not just a code rule.

Every public chart function in `charts/` MUST conform to this signature contract:

```python
def generate_{chart_name}(...) -> Optional[str]:
    """Returns base64 PNG string, or None on any exception."""
    try:
        ...
        return base64_string
    except Exception as e:
        logger.error(f"Chart {chart_name} failed: {e}")
        return None
```

The caller (reporter) checks for `None` and substitutes the fallback HTML string from the Copywriting Contract above. The pipeline never raises from chart generation.

The PEA table function returns `Optional[str]` (HTML string or `None`) following the same contract, even though it does not produce a PNG.
